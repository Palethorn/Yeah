Preparing Application

Project requires several required files following the project structure provided in first steps. Code samples are provided and explained in this section which will get our sample project easily and quickly started.

Modify your composer.json to autoload necessary files. This will ensure that your controllers, models, repositories, and other various files are loaded without any special care.


{
    "require": {
        "palethorn/yeah": "dev-master"
    },
    "autoload": {
        "psr-4": {
            "Blog\\": "blog/src"
        }
    }
}

Create application class. You will want to override certaing methods down the line to configure services, load routes, handle extra configuration:


    // blog/src/Blog.php
    namespace Blog;

    class Blog extends \Yeah\Fw\Application\App {

    }

Finally, create index.php. Include composer autoloader, create initial configuration and initialize your application.


    // web/index.php

    // Include composer autoloader
    require_once '../vendor/autoload.php';

    $config = array(
        'dev' => array()
    );

    $app = new Blog('blog', 'dev', $config);

    $app->execute();

index.php is the application entry point. From there App::execute takes place, which handles request initialization, routing, parameter parsing etc.
