First Steps

Installation

Easiest way to install is to use composer. Create your project folder, add composer.json :


{
    "require": {
        "palethorn/yeah": "dev-master"
    }
}
Then: 

composer update



You have a single folder called vendor inside your project folder. Inside of it is everything you need to get started.
You have installed Yeah! PHP Framework. We can refer to it as YeahFW.



Creating project structure



We'll start by building a sample application called blog. We need to create a project structure for it. YeahFW accepts
whatever structure you impose on it, as long as it's configured correctly. Sample structure we can use for our sample
project is this:




.
├── blog
│   ├── config
|   ├── src
│   │   │── Controllers
|   │   │   └── HomeController.php
|   │   │── Models
│   │   │── Repositories
|   |   └── Blog.php
│   ├── data
│   └── views
│       ├── index.php
│       └── layouts
│           └── default.php
│── vendor
│── web
│    └── app_name.php
└── composer.json





Examplanation of various files and directories:

    
    
  • blog - Holds application specific configuration, controllers, models, and views. Controllers and models
can be namespaced
    • 
    
  • config - Holds various app configuration. For example: paths.php can hold various project paths, db.php holds
database specific configuration etc.
    • 
    
  • controllers - Put application controllers here. Controllers hold actions which are executed for specific
URL
    • 
    
  • data - Various init files. For example: database skeletons, fixtures, export/import files and similar.
    • 
    
  • models - Application ORM model files go here.
    • 
    
  • views - Application views which are rendered in ther respective actions go here.
    • 
    
  • Blog.php - Your main application class. Application is instantiated using this class. It holds all app
initialization specifics.
    • 
    
  • vendor - Composer* packages go here. Everything you installed using composer is residing in this folder and is
managed through composer. Best not to touch it. Most of it is autogenerated and can be overwritten when composer
update is run.
    • 
    
  • web - All publicly accessible files go here. PHP index files, javascript files, stylesheets, static html files
go here.
    • 
    
  • composer.json - Exposes project dependencies which are managed via composer*.
    •
